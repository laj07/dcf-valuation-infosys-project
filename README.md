# DCF Valuation Model - Infosys Limited (NSE: INFY)

A Python implementation of a five-year Discounted Cash Flow (DCF) valuation model for Infosys Limited. The tool fetches audited financial data via the `yfinance` API, projects Free Cash Flow to Firm (FCFF) across a five-year forecast horizon, computes the Weighted Average Cost of Capital (WACC) using the Capital Asset Pricing Model (CAPM), and derives an implied intrinsic value per share. Output includes a full valuation bridge and a two-dimensional sensitivity analysis across WACC and terminal growth rate assumptions.

Built as an independent finance and programming portfolio project. Data sourced from Yahoo Finance and Infosys Annual Reports (FY2021–FY2024).

---

## Background

Discounted Cash Flow analysis is a foundational valuation methodology in investment banking, equity research, and corporate finance. The core premise is that the value of any asset equals the present value of the cash flows it is expected to generate over its life, discounted at a rate that reflects the risk of those cash flows.

This project applies that methodology to Infosys Limited, India's second-largest IT services company by revenue, listed on the NSE (INFY) and NYSE (INFY). Infosys was selected for its large-cap stability, publicly available audited financials, and relevance to both Indian and global technology sector analysis.

## Methodology

### 1. Free Cash Flow to Firm (FCFF)

Free Cash Flow to Firm represents the cash generated by a company's operations available to all capital providers (both equity and debt holders), after accounting for capital expenditure and working capital requirements. It is computed as:

```
FCFF = NOPAT + D&A − CapEx − ΔNWC

where:
  NOPAT  = EBIT × (1 − Effective Tax Rate)
  D&A    = Depreciation and Amortisation (non-cash; added back)
  CapEx  = Capital Expenditure
  ΔNWC   = Change in Net Working Capital
```

NOPAT (Net Operating Profit After Tax) is used rather than net income to keep the metric capital-structure neutral — it reflects operating performance independent of how the firm is financed.

### 2. Revenue and Margin Projections

Historical revenue growth rates and EBIT margins are computed from four years of audited financials (FY2021–FY2024). The projected growth rate is the four-year compound average, capped at 12% to avoid extrapolating cyclically high growth into perpetuity. CapEx and D&A are projected as a percentage of revenue using their respective historical averages.

### 3. WACC (Weighted Average Cost of Capital)

WACC is the blended required rate of return across all of a company's capital providers, weighted by their proportional share of total capital. It serves as the discount rate for FCFF.

```
WACC = (E/V) × Ke  +  (D/V) × Kd × (1 − Tax Rate)

where:
  E   = Market value of equity
  D   = Market value of debt
  V   = E + D
  Ke  = Cost of equity (CAPM)
  Kd  = Pre-tax cost of debt

Cost of Equity (CAPM):
  Ke  = Rf + β × ERP
  Rf  = Risk-free rate (India 10-year G-Sec yield, ~6.85%)
  β   = Equity beta (NSE 5-year monthly, ~0.75 for INFY)
  ERP = Equity Risk Premium (Damodaran India estimate, ~6.5%)
```

The after-tax cost of debt reflects the tax deductibility of interest payments. Infosys carries minimal financial debt relative to its market capitalisation, so WACC is driven almost entirely by the cost of equity.

### 4. Terminal Value

A five-year explicit forecast cannot capture the full value of a going concern. Terminal Value accounts for all cash flows beyond the forecast horizon, assuming the business grows at a stable, long-run rate in perpetuity. The Gordon Growth Model is applied:

```
Terminal Value = FCFF₅ × (1 + g) / (WACC − g)

where:
  g = Terminal growth rate (4.0%; anchored below India's long-run nominal GDP growth)
```

The terminal value is then discounted to present value using the same WACC.

### 5. Intrinsic Value per Share

```
Enterprise Value  =  PV of FCFF (Years 1–5)  +  PV of Terminal Value
Equity Value      =  Enterprise Value − Net Debt
Intrinsic Price   =  Equity Value / Shares Outstanding
```

### 6. Sensitivity Analysis

A two-dimensional sensitivity table is produced varying WACC (±200 basis points around the base case) and the terminal growth rate (2%–6%). This illustrates the range of plausible intrinsic values and highlights which assumptions drive the output most materially.

## Limitations

This model is a mechanistic application of standard DCF methodology and carries the following limitations:

- **Backward-looking inputs.** Projections are anchored to historical averages and do not incorporate forward guidance, analyst consensus estimates, or qualitative assessments of strategic positioning.
- **No scenario analysis beyond sensitivity.** The model does not model bear/base/bull cases with differentiated assumptions for each.
- **Currency risk excluded.** Approximately 60% of Infosys's revenue is denominated in USD. INR/USD fluctuations are not modelled.
- **Terminal value sensitivity.** Terminal value accounts for approximately 73% of enterprise value at base case assumptions — a pattern typical of mature, stable businesses. This makes the model highly sensitive to the terminal growth rate and WACC, as visible in the sensitivity table.
- **No comparable company cross-check.** In practice, a DCF should be triangulated against trading multiples (EV/EBITDA, P/E) derived from comparable listed companies (TCS, Wipro, HCL Technologies, Accenture) and precedent M&A transaction analysis.

## Installation and Usage

**Requirements:** Python 3.8 or higher.

```bash
git clone https://github.com/yourusername/dcf-valuation-infosys.git
cd dcf-valuation-infosys
pip install -r requirements.txt
python dcf_valuation.py
```

The model first attempts to fetch live data from Yahoo Finance via `yfinance`. If the API is unavailable, it falls back automatically to hardcoded FY2021–FY2024 figures sourced from Infosys Annual Reports.

To adapt the model for a different company, update the `CONFIG` dictionary at the top of `dcf_valuation.py` with the relevant ticker, name, and jurisdiction-appropriate risk-free rate. Replace `get_fallback_data()` with audited figures from the target company's annual reports.
